# Implementation Plan: Archive Fallback and Kanban Visibility

This plan addresses the issue where archived plans are inaccessible in the Ticket View, and adds management controls for completed cards on the Kanban board.

## User Review Required

> [!IMPORTANT]
> **Archive Fallback**: The extension will now "see through" the archive. If a plan file is missing from the main `.switchboard/plans/` folder, it will automatically check `.switchboard/archive/plans/`. This prevents "Plan file not found" errors for completed work.

> [!NOTE]
> **Kanban Visibility**: I will add a "Show Completed" toggle to the Kanban board header. 
> - If **OFF** (default), the "COMPLETED" column will be hidden to keep the board focused.
> - If **ON**, the "COMPLETED" column will appear with your history of finished work.
> This preference will be saved in your VS Code settings.

## Proposed Changes

### 1. Plan File Resolution (Backend)

#### [MODIFY] [TaskViewerProvider.ts](file:///d:/aitools/switchboard-mine/switchboard/src/services/TaskViewerProvider.ts)
-   Update `_getPlanPathFromSheet` to implement a fallback mechanism.
-   Logic:
    1. Check primary path (e.g., `.switchboard/plans/my-plan.md`).
    2. If not found, check fallback path (`.switchboard/archive/plans/my-plan.md`).
    3. Return the first one that exists, or the primary path if both are missing (preserving existing error reporting).

### 2. Kanban Orchestration (Backend)

#### [MODIFY] [KanbanProvider.ts](file:///d:/aitools/switchboard-mine/switchboard/src/services/KanbanProvider.ts)
-   Add `_showCompleted` state, initialized from VS Code configuration `switchboard.kanban.showCompleted`.
-   Update `_refreshBoardImpl` to respect `_showCompleted`.
-   Handle a new message `setShowCompleted` from the webview to update the configuration and refresh the board.

### 3. Kanban UI (Frontend)

#### [MODIFY] [kanban.html](file:///d:/aitools/switchboard-mine/switchboard/src/webview/kanban.html)
-   Add a toggle switch in the header (near the workspace selector).
-   Update the JavaScript to send the toggle state to the extension.
-   Ensure the "COMPLETED" column respects the visibility state.

### 4. Configuration (Metadata)

#### [MODIFY] [package.json](file:///d:/aitools/switchboard-mine/switchboard/package.json)
-   Add `switchboard.kanban.showCompleted` (boolean, default: `false`) to `contributes.configuration`.

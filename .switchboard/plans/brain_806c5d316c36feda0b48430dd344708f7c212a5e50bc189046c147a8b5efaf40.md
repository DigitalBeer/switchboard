# Fix Kanban UI Alignment and Labeling

This plan addresses several UI inconsistencies in the Kanban board:
1.  Renaming the "Created" column to "Plan Created".
2.  Fixing vertical alignment of column headers.
3.  Fixing vertical alignment of cards by ensuring the auto-move bar area is consistent across all columns.

## Proposed Changes

### Kanban Webview

#### [MODIFY] [kanban.html](file:///C:/Users/patvu/Documents/GitHub/switchboard/src/webview/kanban.html)

- **Unify Header Heights**:
    - Update `.column-header` to have a consistent `min-height` (e.g., 52px).
    - Set `.column-agent` and `.column-subline-spacer` to have a consistent `height` (e.g., 14px) and `margin-top` to prevent layout shifts when agent names are empty.
    - Ensure `.column-name` has a consistent `line-height`.
- **Rename Column**:
    - Change "Created" to "Plan Created" in the header of the first column.
- **Unify Auto-move Bars**:
    - Remove the hidden content from the `CODE REVIEWED` column's auto-move bar.
    - Ensure `.auto-move-bar` has a consistent `min-height` so it takes space even when empty.
    - Remove `invisible-placeholder` class and instead just keep the bar empty but styled.

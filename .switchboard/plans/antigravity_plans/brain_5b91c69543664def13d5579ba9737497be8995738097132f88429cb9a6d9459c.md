## Proposed Changes

### Component: Switchboard Webview Backend

#### [MODIFY] [TaskViewerProvider.ts](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)

**Fix A (Required):** In [_handleAirlockSendToCoder](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts#6240-6283) (line ~6269), remove `${text}` from the payload entirely. The dispatched message must only contain the instruction text and the absolute `patchPath`. No raw content from the textarea.

```
// BEFORE:
const payload = `...Patch file: ${patchPath}\n\n${text}`;

// AFTER:
const payload = `This is a patch from the Airlock. Please manually apply the patch file:\n\n${patchPath}\n\nUse the \`apply_patch\` skill or read the file and apply changes manually.`;
```

**Fix B (Required):** Add a `MAX_AIRLOCK_TEXT_BYTES = 2 * 1024 * 1024` (2MB) guard in both [_handleAirlockSendToCoder](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts#6240-6283) and [_handleAirlockConvertToPlan](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts#6213-6239) before the [writeFile](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts#5537-5543) call. Return `airlock_coderError` / `airlock_planError` to the webview if exceeded.

**Fix E (Recommended):** Swap dispatch priority to `leadAgent || coderAgent` (Lead Coder gets Airlock patches first).

### Component: Switchboard Webview Frontend

#### [MODIFY] [implementation.html](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/webview/implementation.html)

**Fix C (Required — Optimistic Clear):** Clear `_airlockTextareaValue = ''` and `textarea.value = ''` **on button click**, *before* posting the message to the backend. Restore value on error via `airlock_planError` / `airlock_coderError`. Do NOT wait for `airlock_planSaved` or `airlock_coderSent` — this avoids the race condition with the plan-watcher-triggered re-render.

**Fix D (Required — Textarea Stabilization):** In [renderAgentList()](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/webview/implementation.html#2035-2122) at line ~2047, change the `agentListWebai` rebuild strategy:
- If `document.getElementById('airlock-textarea')` exists AND the `webai` sub-tab is currently active (`currentAgentTab === 'webai'`), **skip** clearing and rebuilding `agentListWebai`. Only update the `webai-status` element via ID if a status message arrives.
- If the webai tab is not active, proceed with normal teardown + rebuild.
This prevents mid-typing DOM destruction.

**Fix F (Recommended — Button Layout):** Wrap `planBtn` and `coderBtn` in a flex row container with `gap: 8px`. Use `flex: 1; min-width: 0;` on each button. Shorten labels to `PLAN` and `SEND TO CODER` if they overflow at standard sidebar widths.

## Verification Plan

### Manual Verification
1. **CLI Limit (Fix A):** Paste 500KB+ text. Click "Send to Coder". The agent terminal should only receive a short instruction string + the file path — no raw text dump.
2. **Size Guard (Fix B):** (If testable) Attempt to submit text exceeding 2MB. Webview should display an error. Textarea should NOT be cleared.
3. **Optimistic Clear (Fix C):** Paste text. Click "Convert to Plan". Textarea immediately clears on click. If the backend throws an error, the text is restored.
4. **Typing Stability (Fix D):** While typing in the airlock textarea, trigger an external state change (rename a terminal, save state.json). Cursor and input value MUST be preserved.
5. **Layout (Fix F):** Verify side-by-side buttons display correctly at both default (240px) and narrow sidebar widths.

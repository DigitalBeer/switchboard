# Coder/Lead Coder Toggle for Kanban "Coded" Column

The user wants to persistently switch between "Coder" and "Lead Coder" as the target agent for cards in the Kanban's "Coded" column.

## Proposed Changes

### [Kanban Board]

#### [MODIFY] [kanban.html](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/webview/kanban.html)
- Add a `<select>` element in the `.column-header` for the "CODED" column.
- Styling: Add CSS for the column-specific dropdown to match the industrial theme.
- Logic:
    - Post a message `setColumnTarget` when the dropdown value changes.
    - Handle an `updateTarget` message from the extension to set the initial/persisted value of the dropdown.

#### [MODIFY] [KanbanProvider.ts](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)
- Add a state variable to track the current target for the "CODED" column (e.g., `_codedColumnTarget`).
- In [_handleMessage](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts#197-224):
    - Handle `setColumnTarget` to update `_codedColumnTarget` and persist it using `this._context.workspaceState` (or similar memento).
- In [_columnToRole](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts#225-236):
    - Instead of hardcoding 'lead' for 'CODED', return `this._codedColumnTarget`.
- In [open()](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts#43-84):
    - Load the persisted target value.
- In [_refreshBoard()](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts#140-155) (or similar):
    - Send the current target to the webview so the dropdown stays in sync.

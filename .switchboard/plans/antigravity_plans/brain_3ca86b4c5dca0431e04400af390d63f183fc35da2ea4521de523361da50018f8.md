# Improve Airlock Context Bundling Logic

## Goal
Transition the Airlock bundling architecture from "Folder-Based Grouping" to a "Rolling Size-Based Chunking" model. This will resolve issues where massive documents cause AI truncation, specifically ensuring raw HTML, CSS, and dotfiles are reliably maintained by capping file chunk sizes down to 500KB parts. **Critically, all context chunk bundles will continue to be generated as `.docx` files** using the existing [saveAsDocx](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/ContextBundler.ts#25-57) logic. The manifest itself will be written as a clean [manifest.md](file:///c:/Users/patvu/Documents/GitHub/switchboard/.switchboard/airlock/manifest.md).

## Proposed Changes

### ContextBundler.ts Core Logic

#### [MODIFY] [ContextBundler.ts](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/ContextBundler.ts)
*   **Constants**: Replace the `MAX_BUNDLE_BYTES` (20MB) and `SIZE_THRESHOLD_BYTES` (500KB) definition block with a strict single constraint: `CHUNK_LIMIT_BYTES = 500 * 1024`.
*   **Remove Helpers**: Delete the [getTopLevelDir](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/ContextBundler.ts#68-72) function entirely, as files will no longer be mapped by their parent folder.
*   **Refactor [bundleWorkspaceContext](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/ContextBundler.ts#73-191)**:
    1.  Remove the `dirSizes` aggregation loop (Step 4 of the old logic).
    2.  Replace the `categoryBuffers` map with a single rolling buffer `currentBuffer` (string) and track its active size via `currentBytes` (number).
    3.  Iterate through `files`. Any file larger than `CHUNK_LIMIT_BYTES` is explicitly logged in `manifestLines` as excluded and skipped.
    4.  If adding the current file to `currentBuffer` exceeds `CHUNK_LIMIT_BYTES` (and the buffer isn't empty), flush the `currentBuffer` using [saveAsDocx(path.join(outputDir, \](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/ContextBundler.ts#25-57)\${timestamp}-bundle-part-\${currentChunkIndex}.docx\`), currentBuffer)`. Increment `currentChunkIndex` and reset the buffer.
    5.  Map each included file to its specific chunk (`...-bundle-part-N.docx`) in the `manifestLines` array, extracting descriptive header comments where possible.
    6.  When the loop concludes, flush any remaining content in `currentBuffer` to the final `.docx` chunk.
    7.  Write the manifest specifically as [.md](file:///c:/Users/patvu/Documents/GitHub/switchboard/.switchboard/airlock/manifest.md): `await fs.promises.writeFile(path.join(outputDir, \`\${timestamp}-manifest.md\`), manifestContent)` containing an array of Markdown bullet points linking files to their chunks (e.g., `- **file.ts** (4.2KB) -> \`bundle-part-1.docx\``).

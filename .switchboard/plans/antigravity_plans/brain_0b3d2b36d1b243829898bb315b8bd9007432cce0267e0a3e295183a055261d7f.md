# Kanban Planner Improvements

The Switchboard Kanban board needs improvements to better reflect the workflow status. The 'Reviewed' column will be renamed to 'Plan Reviewed', and the logic for deriving the column from runsheet events will be updated to ensure plans sent to the planner automatically start in this column.

## Proposed Changes

### Webview UI

#### [MODIFY] [kanban.html](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/webview/kanban.html)
- Rename the display label for the `REVIEWED` column to `PLAN REVIEWED`.
- Update internal constants and element IDs to use `PLAN REVIEWED` for consistency with the backend.

---

### Backend Services

#### [MODIFY] [KanbanProvider.ts](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)
- Update [KanbanColumn](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts#7-8) type definition to replace `REVIEWED` with `PLAN REVIEWED`.
- Enhance [_deriveColumn](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts#182-200) logic to recognize `sidebar-review` and `Enhanced plan` as triggers for the `PLAN REVIEWED` column.
- Update [_columnToRole](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts#236-247) mapping to correctly route `PLAN REVIEWED` actions to the `planner` agent.

# Fully Autonomous Agentic CI/CD Implementation Plan

This plan is explicitly designed to be executed by an AI LLM Assistant. The goal is to evolve the Switchboard protocol from a "Human-In-The-Loop" architecture to a fully autonomous Multi-Agent swarm that requires zero manual intervention from the 'scrum master' or 'tester' roles.

## User Review Required

> [!IMPORTANT]
> The current state has been committed, and we are safely operating on the newly created `autonomous-agents-v2` branch. Review this plan and approve it to have an agent begin execution.

## Architectural Changes Needed

Currently, agents stop executing and say "Please verify this is correct" (the "Yield-Based" completion protocol). We are fundamentally replacing this with:
1. **Adversarial QA Gate**: Instead of asking the user, the 'Coder' submits to a 'QA Agent' that runs tests and loops back to a 'Fixer' on failure.
2. **Automated Handoffs**: Agents will automatically use `move_kanban_card` to trigger the next phase.

### 3. The Autonomous QA Tester Pipeline

This replaces the human tester.

#### [NEW] [.agent/workflows/qa-pipeline.md](file:///d:/aitools/switchboard-mine/switchboard/.agent/workflows/qa-pipeline.md)
*   **Instruction for AI**: Create this workflow to serve as the adversarial quality gate.
*   **Content Required**:
    1. Define the trigger `trigger: /qa-pipeline`.
    2. Instruct the agent to read the current implementation plan and the Git diff.
    3. Instruct the agent to forcibly run `npm run test` or `npm run build` using the terminal tool.
    4. **Decision Tree Logic**:
        *   **IF SUCCESS (Exit Code 0)**: Call `move_kanban_card(sessionId, 'DONE')`.
        *   **IF FAILURE (Exit Code > 0)**: Append the terminal error to the current plan's `.md` file under a `# QA Fixer Note` header. Move the Kanban card back to `CODED` to re-trigger the coder agent. Provide strict loop limits (e.g., maximum 3 failures before pausing).
